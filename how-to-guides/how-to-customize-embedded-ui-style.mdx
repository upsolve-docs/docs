Upsolve has built-in design system that makes the application adoptable with different themes and views. This design system helps you to create different themes with using different colors and sizes. With this flexibility you can customize the embedded Upsolve's view and make its look and feel as same as your application.

Upsolve design system is built to offer flexibility through CSS variables, allowing you to adjust and personalize the appearance of your application easily.

Here is the reference of the customizable variables.

# CSS Reference

## General Colors

There are two generic color codes in Upsolve's Design System. Most of the UI components use these color options.

General colors have 5 different variants per class as `base`, `hover`, `dark`, `light`, `lighter`

### Primary Colors

You can change the primary colors of the Upsolve with overriding following css variables

```css
--primary-base: #5f62f0;
--primary-hover: #5457cd;
--primary-dark: #363885;
--primary-light: #bcbdf9;
--primary-lighter: #dadafc;
```

### Secondary Colors

You can override the secondary colors of the Upsolve with following css variables.

```css
--secondary-base: #9aa4be;
--secondary-hover: #444d66;
--secondary-dark: #2d354c;
--secondary-light: #e3e6eb;
--secondary-lighter: #eef0f3;
```

## Severity Colors

There are 6 severity option in Upsolve's Design System. These are `Success`, `Error`, `Warning`, `Informative`, `Neutral` and `Updating`.

Each severity option has 5 different variants like general colors.

### Success Colors

Success colors are used for express result for successful activities or harmless actions.

```css
--success-base: #22c55e;
--success-hover: #16a34a;
--success-dark: #15803d;
--success-light: #bbf7d0;
--success-lighter: #dcfce7;
```

### Warning Colors

Warning colors are used in components that the user should be warn.

```css
--warning-base: #f97316;
--warning-hover: #ea580c;
--warning-dark: #c2410c;
--warning-light: #fed7aa;
--warning-lighter: #ffedd5;
```

### Error Colors

Error colors are used for errors and actions that user should reconsider the action.

```css
--error-base: #ef4444;
--error-hover: #dc2626;
--error-dark: #b91c1c;
--error-light: #fecaca;
--error-lighter: #fee2e2;
```

### Informative Colors

These colors are used for informations

```css
--information-base: #3b82f6;
--information-hover: #326fd1;
--information-dark: #295bac;
--information-light: #abc9fb;
--information-lighter: #d0e1fd;
```

### Neutral Colors

These are also used for informations. These can be thought as secondary for iformative colors

```css

--neutral-hover: #444d66;
--neutral-dark: #2d354c;
--neutral-light: #e3e6eb;

```

### Updating Colors

These colors are used in ongoing processes in Upsolve. Such as caching process.

```css

--updating-dark: #0b655b;
--updating-light: #9ae0d9;
```

Upsolve uses 5 different background token in its Design System. You can override these tokens to implement your own look and feel

```css

--background-secondary: #1d1e1f;
```
